    Reference¶

    CLI¶

    Command:

    nomad-plugin-metadata extract [options]

    Options:

    • --repo-path (default: .)
    • --manual-path (default: .metadata/nomad_plugin_metadata.manual.yaml)
    • --auto-path (default: .metadata/nomad_plugin_metadata.auto.yaml)
    • --effective-path (default: nomad_plugin_metadata.yaml)
    • --report-path (default: .metadata/plugin-metadata.override-report.yaml)
    • --release-tag (default: empty)
    • --release-sha (default: empty)
    • --create-manual-template-if-missing / --no-create-manual-template-if-missing

    Extractor behavior notes:

    • Baseline metadata is read from pyproject.toml.
    • People metadata uses CITATION.cff / citation.cff authors first when present, then falls back to pyproject.toml authors.
    • Maintainers are read from pyproject.toml (project.maintainers).
    • Technical metadata is additionally read from installed nomad.plugin entry points (for example parser matcher fields such as mainfile_*, parser level/aliases, dict matcher, and supported compressions).
    • supported_filetypes and file_format_support are inferred from parser mainfile_name_re and concrete auxiliary file patterns where present.
    • GitHub repository telemetry is fetched best-effort from repository API: stars, owner, owner_type, created, last_updated, archived.
    • If plugin entry points are not installed/importable, extraction falls back to static pyproject entry-point declarations.
    • Maturity heuristic:
    • archived if GitHub repository is archived (best-effort GitHub API lookup)
    • otherwise stable for versions >=1.0.0
    • manual override in .metadata/nomad_plugin_metadata.manual.yaml still takes precedence.
    • URL inference:
    • upstream_repository: project.urls.Repository first, otherwise CITATION.cff repository-code.
    • documentation: project.urls.Documentation first, otherwise best-effort GitHub Pages URL (https://<owner>.github.io/<repo>/) if reachable. CITATION.cff url is intentionally not used here.
    • homepage: project.urls.Homepage first, otherwise CITATION.cff url, otherwise resolved upstream_repository.

    Metadata files¶

    • Manual template/override file:
    • .metadata/nomad_plugin_metadata.manual.yaml
    • created with inline comments describing expected values/types/enums
    • contains manual-primary and conditional fields; copy autogenerated fields from auto into manual only when overriding.
    • Auto-generated file:
    • .metadata/nomad_plugin_metadata.auto.yaml
    • Effective merged/query file:
    • nomad_plugin_metadata.yaml
    • Override report:
    • .metadata/plugin-metadata.override-report.yaml

    Reset behavior¶

    • Deleting .metadata/* resets manual/auto/report artifacts.
    • Deleting .metadata/nomad_plugin_metadata.auto.yaml, .metadata/plugin-metadata.override-report.yaml, and nomad_plugin_metadata.yaml resets autogenerated artifacts while keeping manual overrides.
    • Deleting .metadata/nomad_plugin_metadata.manual.yaml additionally resets manual curation (template will be recreated on next extract run).
